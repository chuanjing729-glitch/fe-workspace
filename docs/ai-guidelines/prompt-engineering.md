# 提示词与指令规范

## 概述

提示词（Prompt）和指令是与AI编码助手有效交互的关键。良好的提示词能够让AI更准确地理解开发者意图，生成更符合需求的代码。本文档旨在为团队成员提供提示词和指令编写的最佳实践规范。

## 提示词基本原则

### 1. 明确性原则
提示词应当清晰明确，避免模糊不清的表述。

```javascript
// ❌ 不明确的提示词
// "创建一个表单"

// ✅ 明确的提示词
// "创建一个用户注册表单，包含用户名、邮箱、密码输入框，以及同意条款的复选框"
```

### 2. 具体性原则
提供具体的细节和要求，包括技术栈、功能需求、约束条件等。

```javascript
// ❌ 不具体的提示词
// "优化这段代码"

// ✅ 具体的提示词
// "优化以下JavaScript函数，减少时间复杂度，当前是O(n²)，目标是O(n log n)，使用快排算法"
```

### 3. 上下文原则
提供必要的上下文信息，帮助AI更好地理解需求背景。

```javascript
/*
 * 项目技术栈：Vue 3 + TypeScript + Element Plus
 * 需求：创建一个商品列表组件
 * 要求：
 * 1. 支持分页功能
 * 2. 支持按价格、销量排序
 * 3. 支持关键词搜索
 * 4. 使用Composition API实现
 */
```

## 高质量提示词要素

### 1. 角色设定
明确AI扮演的角色，有助于获得更专业的回答。

```
"你是一位资深的前端架构师，精通Vue和React..."
```

### 2. 任务描述
清楚地描述需要完成的任务。

```
"请帮我实现一个防抖函数..."
```

### 3. 技术要求
指定使用的技术栈、框架或库。

```
"使用Vue 3 Composition API和TypeScript实现..."
```

### 4. 格式要求
指定输出格式，如代码、表格、列表等。

```
"请以代码形式回复，并添加必要的注释"
```

### 5. 约束条件
说明任何限制或特殊要求。

```
"不要使用第三方库，只使用原生JavaScript"
```

## 指令编写规范

### 1. 指令结构
推荐使用以下结构编写指令：

```
角色设定 + 任务描述 + 技术要求 + 格式要求 + 约束条件
```

### 2. 指令示例

#### 代码生成指令
```
你是一位经验丰富的前端开发工程师，熟悉Vue 3和TypeScript。
请创建一个用户信息展示组件，要求：
1. 使用Vue 3 Composition API
2. 使用TypeScript编写
3. 包含用户头像、姓名、邮箱、注册时间等信息
4. 支持编辑功能
5. 以单文件组件(.vue)格式返回代码
6. 不要使用任何第三方UI库
```

#### 代码优化指令
```
你是一位性能优化专家。
请分析以下代码的性能问题，并提供优化建议：
[代码内容]
要求：
1. 指出具体的性能瓶颈
2. 提供优化后的代码
3. 说明优化原理
4. 以列表形式组织回答
```

#### 错误修复指令
```
你是一位调试专家。
以下JavaScript代码存在错误，请找出并修复：
[错误代码]
要求：
1. 准确指出错误位置和原因
2. 提供修复后的代码
3. 解释为什么会出现这个错误
4. 给出避免类似错误的建议
```

## 常见场景提示词模板

### 1. 组件开发模板
```
你是一位前端开发专家，熟悉[技术栈]。
请创建一个[组件名称]组件，要求：
1. 技术栈：[Vue/React/Angular] + [TypeScript/JavaScript]
2. 功能需求：[具体功能列表]
3. 样式要求：[CSS框架/样式规范]
4. 交互要求：[事件处理、动画等]
5. 输出格式：完整的单文件组件代码
6. 其他要求：[特殊约束]
```

### 2. 工具函数模板
```
你是一位函数式编程专家。
请编写一个[函数功能]的工具函数，要求：
1. 函数签名：[参数类型和返回值类型]
2. 功能描述：[详细功能说明]
3. 边界条件处理：[特殊情况处理]
4. 性能要求：[时间/空间复杂度]
5. 输出格式：可直接运行的JavaScript/TypeScript代码
```

### 3. API调用模板
```
你是一位后端接口专家。
请封装一个调用[API名称]的函数，要求：
1. 使用[axios/fetch/其他]实现
2. 包含错误处理和loading状态
3. 支持[GET/POST/PUT/DELETE]方法
4. 参数验证和类型检查
5. 输出格式：完整的函数代码和使用示例
```

### 4. 代码审查模板
```
你是一位代码审查专家，熟悉[技术栈]最佳实践。
请审查以下代码并提出改进建议：
[代码内容]
审查要点：
1. 代码规范和风格
2. 性能优化建议
3. 安全性检查
4. 可维护性评估
5. 最佳实践应用
```

## 提示词优化技巧

### 1. 迭代优化
如果初次提示词效果不佳，可以通过以下方式进行优化：
- 增加更多细节描述
- 明确技术要求
- 提供参考示例
- 指出不满意的地方并要求改进

### 2. 分解复杂任务
对于复杂需求，可以分解为多个简单提示词：

```
第一步：设计数据结构
第二步：实现核心算法
第三步：编写UI组件
第四步：整合完整功能
```

### 3. 使用示例引导
提供参考示例可以提高AI输出的准确性：

```
请参考以下组件的结构和风格，创建一个新的组件：
[示例组件代码]
新组件的要求是：[新组件需求]
```

## 避免的常见错误

### 1. 过于简略的提示词
```javascript
// ❌ 太简单
// "写一个函数"

// ✅ 更具体
// "用JavaScript写一个防抖函数，延迟300ms，支持立即执行选项"
```

### 2. 缺乏上下文信息
```javascript
// ❌ 缺少上下文
// "优化这个组件"

// ✅ 提供上下文
/*
 * 项目使用Vue 2，需要优化以下组件的渲染性能
 * 主要问题是列表数据过多导致页面卡顿
 * [组件代码]
 */
```

### 3. 不切实际的要求
```javascript
// ❌ 不现实
// "用一行代码实现所有功能"

// ✅ 合理要求
// "简化以下代码，减少重复逻辑，提高可读性"
```

### 4. 忽略约束条件
```javascript
// ❌ 忽略限制
// "创建一个图片处理功能"

// ✅ 明确约束
// "创建一个图片压缩功能，要求：
// 1. 只使用原生JavaScript
// 2. 支持JPEG/PNG格式
// 3. 压缩率可调节(50%-90%)
// 4. 保持图片宽高比"
```

## 最佳实践总结

### 1. 建立团队提示词库
- 收集和整理常用的高质量提示词
- 按场景分类管理
- 定期更新和完善

### 2. 培训与分享
- 组织提示词编写培训
- 分享成功的提示词案例
- 讨论失败的提示词教训

### 3. 持续改进
- 收集使用反馈
- 分析效果数据
- 优化提示词模板

通过遵循这些规范和技巧，团队成员可以更有效地使用AI编码助手，提高开发效率和代码质量。
