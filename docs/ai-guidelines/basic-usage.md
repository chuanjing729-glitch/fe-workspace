# 基础使用技巧

## 智能代码补全

### 触发方式
- 自然编码停顿：在编写代码时稍作停顿，灵码会自动给出补全建议
- 快捷键触发：使用 `Ctrl+Space`（Windows/Linux）或 `Cmd+Space`（Mac）手动触发

### 最佳实践
1. **函数定义补全**
   ```javascript
   // 在定义函数时稍作停顿
   function calculateTotalPrice(items) {
     // 灵码会提供相关的方法和属性建议
   }
   ```

2. **变量声明补全**
   ```javascript
   // 声明变量时描述意图
   const userProfile = // 灵码会根据上下文提供初始化建议
   ```

3. **对象属性补全**
   ```javascript
   const user = {
     name: 'John',
   ```

## 代码解释

### 使用场景
- 理解第三方库代码
- 分析遗留代码
- 学习复杂逻辑实现

### 操作方式
1. 选中需要解释的代码片段
2. 右键选择"解释代码"功能
3. 查看灵码生成的详细解释

### 解释内容
- 代码功能概述
- 关键逻辑说明
- 输入输出说明
- 可能的优化建议

## 快捷键汇总

| 功能 | Windows/Linux | Mac | 说明 |
|------|---------------|-----|------|
| 触发补全 | Ctrl+Space | Cmd+Space | 手动触发代码补全 |
| 接受建议 | Tab | Tab | 接受当前建议 |
| 拒绝建议 | Esc | Esc | 取消当前建议 |
| 解释代码 | Ctrl+Shift+E | Cmd+Shift+E | 解释选中代码 |
| 生成注释 | Ctrl+Shift+D | Cmd+Shift+D | 为函数生成注释 |
| 代码优化 | Ctrl+Shift+O | Cmd+Shift+O | 优化选中代码 |

## 使用技巧

### 1. 提高补全准确性
- 在注释中清晰描述意图

- 明确指定技术栈和框架

### 3. 提升使用效率
- 熟练掌握快捷键
- 了解工具的能力边界
