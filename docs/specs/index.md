# 开发规范总览

欢迎来到前端工程效率平台的开发规范文档。本规范旨在提升团队协作效率、保证代码质量和项目可维护性。

## 📝 代码编写规范

规范日常代码编写的风格、命名、注释等，确保代码的一致性和可读性。

- [命名规范](/specs/coding/naming) - 文件、变量、函数、组件的命名约定
- [JavaScript/TypeScript 规范](/specs/coding/javascript) - JavaScript 和 TypeScript 编码标准
- [Vue 组件规范](/specs/coding/vue) - Vue 2/3 组件开发规范
- [CSS 样式规范](/specs/coding/css) - CSS/SCSS 编写规范
- [注释规范](/specs/coding/comments) - 代码注释和文档注释标准
- [边界处理规范](/specs/coding/boundary-specification) - 边界条件处理规范
- [空指针防护规范](/specs/coding/null-safety-specification) - 空指针防护规范
- [事件规范](/specs/coding/event-specification) - 事件处理规范
- [NPM 包规范](/specs/coding/npm-package) - NPM 包开发规范

## 🔧 工程化规范

规范项目的工程化流程，包括版本控制、代码审查、构建部署等。

- [Git 工作流](/specs/engineering/git-workflow) - 分支管理、提交规范、合并策略
- [Code Review 规范](/specs/engineering/code-review) - 代码审查流程和清单
- [构建与部署](/specs/engineering/build-deploy) - 构建配置、部署流程、CI/CD
- [测试规范](/specs/engineering/testing) - 单元测试、集成测试、E2E测试

## 🎨 设计规范

规范技术方案设计、架构设计等技术文档的编写。

- [技术方案设计规范](/specs/design/technical-design) - 技术方案设计流程和原则
- [技术设计文档模板](/specs/design/technical-document) - 标准化的技术文档模板
- [架构设计规范](/specs/design/architecture) - 系统架构、微前端架构设计

## ⚡ 性能优化规范

规范性能优化、内存管理、资源处理等最佳实践。

- [性能优化](/specs/optimization/performance) - 首屏加载、运行时性能优化
- [内存管理](/specs/optimization/memory) - 内存泄漏预防和优化
- [资源处理](/specs/optimization/assets) - 图片、字体等静态资源处理

## 🚀 快速开始

如果你是新成员，建议按以下顺序阅读：

1. [命名规范](/specs/coding/naming) - 了解基本的命名约定
2. [Git 工作流](/specs/engineering/git-workflow) - 了解团队的协作流程
3. [Code Review 规范](/specs/engineering/code-review) - 了解代码审查标准
4. 根据技术栈阅读对应的语言规范（JavaScript、Vue、CSS等）
