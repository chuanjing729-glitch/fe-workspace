core-utils 需求文档 ​

1. 概述 ​

1.1 目的 ​

本文档旨在定义 core-utils 工具库的需求，该工具库提供前端开发中常用的通用工具函数，提高开发效率和代码质量。

1.2 范围 ​

本工具库包含数组处理、字符串处理、对象操作、日期处理、DOM操作、事件处理、表单验证、HTTP请求等常用工具函数。

2. 功能需求 ​

2.1 核心功能模块 ​

2.1.1 数组处理工具 ​

  • 数组去重：支持多种去重策略（全等比较、属性比较等）
  • 数组扁平化：支持指定层级的扁平化处理
  • 数组分组：根据指定条件对数组元素进行分组
  • 数组排序：支持自定义比较函数的排序
  • 数组查找：支持条件查找和索引查找
  • 数组差集、交集、并集运算
  • 数组随机打乱和随机采样

2.1.2 字符串处理工具 ​

  • 字符串格式化：支持模板字符串和参数替换
  • 字符串截取：支持安全截取和按字节截取
  • 字符串验证：支持邮箱、手机号、URL等常见格式验证
  • 字符串转换：支持驼峰命名、下划线命名等转换
  • 字符串编码/解码：支持Base64、URL编码等
  • 字符串相似度计算
  • 字符串首字母大写、全大写、全小写转换

2.1.3 对象操作工具 ​

  • 对象深拷贝：支持循环引用处理
  • 对象合并：支持深度合并和自定义合并策略
  • 对象属性操作：支持属性获取、设置、删除
  • 对象验证：支持类型检查和结构验证
  • 对象路径操作：支持嵌套属性的获取和设置
  • 对象映射和过滤
  • 对象转查询参数、查询参数转对象

2.1.4 日期处理工具 ​

  • 日期格式化：支持多种日期格式
  • 日期计算：支持加减天数、月份、年份等
  • 日期比较：支持日期大小比较和差值计算
  • 时区处理：支持时区转换和本地化显示
  • 日期范围计算
  • 工作日计算
  • 日期解析（支持多种格式）

2.1.5 DOM操作工具 ​

  • DOM元素选择：支持CSS选择器和ID选择
  • DOM元素操作：支持创建、插入、删除、克隆元素
  • DOM事件绑定：支持事件监听和解绑
  • DOM样式处理：支持样式获取和设置
  • DOM尺寸和位置计算
  • DOM遍历和查找
  • DOM动画支持

2.1.6 事件处理工具 ​

  • 事件绑定与解绑：支持命名空间和批量操作
  • 事件委托：支持冒泡和捕获阶段
  • 防抖节流：支持自定义延迟时间和立即执行
  • 自定义事件：支持事件触发和监听
  • 事件总线：支持全局事件通信
  • 事件修饰符处理

2.1.7 表单验证工具 ​

  • 基础验证规则：必填、长度、数值范围等
  • 自定义验证规则：支持正则表达式和函数验证
  • 表单数据收集：支持序列化和反序列化
  • 错误信息处理：支持国际化和自定义错误信息
  • 异步验证支持
  • 表单状态管理
  • 验证规则组合

2.1.8 HTTP请求工具 ​

  • GET/POST/PUT/DELETE请求：支持所有HTTP方法
  • 请求拦截器：支持请求前处理和响应后处理
  • 响应拦截器：支持统一错误处理和数据转换
  • 错误处理：支持超时、网络错误等异常处理
  • 请求取消和超时控制
  • 请求缓存机制
  • 文件上传和下载支持

3. 非功能需求 ​

3.1 性能要求 ​

  • 工具函数执行效率高：单个函数执行时间不超过10ms（普通数据量）
  • 内存占用合理：避免内存泄漏，单次调用内存增长不超过1MB
  • 支持按需加载：支持Tree-shaking，仅打包使用的函数
  • 支持缓存机制：对于计算密集型函数提供结果缓存
  • 支持异步处理：对于耗时操作提供异步版本

3.2 兼容性要求 ​

  • 浏览器兼容性：支持Chrome、Firefox、Safari、Edge最新两个版本
  • IE兼容性：支持IE11及以上版本
  • Node.js兼容性：支持Node.js 12及以上版本
  • TypeScript兼容性：提供完整的类型定义文件
  • 框架兼容性：与React、Vue、Angular等主流框架兼容

3.3 安全性要求 ​

  • 避免XSS攻击：所有DOM操作进行HTML转义处理
  • 数据验证和过滤：输入参数进行类型检查和合法性验证
  • 安全的DOM操作：避免直接innerHTML操作，使用安全的DOM API
  • CSRF防护：HTTP请求工具支持CSRF令牌处理
  • 内容安全策略：遵守CSP规范，避免内联脚本

3.4 可维护性要求 ​

  • 代码注释：所有公共API提供详细的JSDoc注释
  • 代码规范：遵循统一的代码风格和命名规范
  • 模块化设计：功能模块解耦，便于独立维护
  • 向后兼容：版本升级保证向后兼容性

3.5 可测试性要求 ​

  • 单元测试覆盖：核心函数单元测试覆盖率达到95%以上
  • 集成测试：提供完整的集成测试用例
  • 测试工具支持：提供测试辅助工具和Mock数据

4. 验收标准 ​

4.1 功能验收标准 ​

  • 所有工具函数功能正确：通过单元测试验证，测试用例覆盖所有边界条件
  • 提供完整的单元测试：每个函数至少包含3个测试用例（正常情况、边界情况、异常情况）
  • 提供详细的使用文档：包含函数说明、参数说明、返回值说明、使用示例
  • 提供类型定义：所有函数提供完整的TypeScript类型定义

4.2 性能验收标准 ​

  • 工具函数执行时间符合预期：单个函数执行时间不超过10ms（普通数据量）
  • 内存泄漏检测通过：使用内存分析工具检测无内存泄漏
  • 打包体积合理：gzip压缩后总体积不超过50KB
  • Tree-shaking效果：仅引入使用的函数，不引入未使用的代码

4.3 兼容性验收标准 ​

  • 浏览器兼容性：在Chrome、Firefox、Safari、Edge最新两个版本中正常运行
  • IE兼容性：在IE11中正常运行
  • Node.js兼容性：在Node.js 12及以上版本中正常运行
  • TypeScript兼容性：在TypeScript 4.0及以上版本中正常使用
  • 框架兼容性：在React、Vue、Angular等主流框架中正常使用

4.4 安全性验收标准 ​

  • XSS防护：所有DOM操作通过安全检查，无XSS漏洞
  • 输入验证：所有函数对输入参数进行验证，防止恶意输入
  • CSP兼容：遵守内容安全策略，无内联脚本违规

4.5 可维护性验收标准 ​

  • 代码质量：通过ESLint等代码检查工具，无严重错误
  • 代码注释：公共API注释完整，符合JSDoc规范
  • 代码复杂度：圈复杂度不超过10

5. 配置选项 ​

5.1 全局配置 ​

  • debug: boolean - 是否开启调试模式，默认false
  • cacheEnabled: boolean - 是否开启缓存，默认true
  • timeout: number - HTTP请求超时时间，默认5000ms

5.2 模块配置 ​

  • dateFormat: string - 默认日期格式
  • domPrefix: string - DOM操作前缀
  • validatorMessages: object - 验证器错误信息配置

6. 与其他模块的协同 ​

core-utils 作为基础工具库，被其他模块依赖使用。

6.1 与 vue2-toolkit 的协同 ​

  • 提供基础工具函数支持
  • 共享配置管理机制

6.2 与 webpack 插件的协同 ​

  • 提供通用工具函数支持
  • 支持在构建时和运行时使用

7. 技术实现要求 ​

7.1 编码规范 ​

  • 使用 TypeScript 编写，提供完整的类型定义
  • 遵循 ESLint 规范，代码风格统一
  • 函数命名采用驼峰命名法，常量采用大写字母加下划线

7.2 模块化设计 ​

  • 采用 ES6 模块化语法
  • 支持按需引入，支持 tree-shaking
  • 每个功能模块独立封装，降低耦合度

7.3 错误处理 ​

  • 统一错误处理机制
  • 提供详细的错误信息和堆栈跟踪
  • 支持自定义错误类型

7.4 性能优化 ​

  • 避免不必要的计算和内存分配
  • 使用 WeakMap 等弱引用避免内存泄漏
  • 对计算密集型函数提供缓存机制

8. 文档要求 ​

8.1 README 文档 ​

  • 提供项目概述和核心功能介绍
  • 包含安装和使用指南
  • 提供详细的 API 文档和使用示例
  • 提供贡献指南和许可证信息

8.2 API 文档 ​

  • 为每个公共函数提供详细的 JSDoc 注释
  • 包含函数说明、参数说明、返回值说明
  • 提供使用示例和注意事项
  • 支持 TypeScript 类型提示

8.3 使用示例 ​

  • 提供完整的使用示例代码
  • 包含常见场景的解决方案
  • 提供最佳实践指导
  • 包含错误处理示例

8.4 更新日志 ​

  • 遵循标准的 CHANGELOG 格式
  • 记录每个版本的功能新增、bug修复和破坏性变更
  • 提供升级指南和兼容性说明

9. 发布和版本管理 ​

9.1 版本规范 ​

  • 遵循语义化版本规范（SemVer）
  • 主版本号：不兼容的API修改
  • 次版本号：向下兼容的功能性新增
  • 修订号：向下兼容的问题修正

9.2 发布流程 ​

  • 自动化测试通过后才能发布
  • 发布前进行代码审查
  • 确保文档和示例代码更新完毕
  • 提供详细的更新日志

9.3 npm 包管理 ​

  • 发布到 npm 公共仓库 (@51jbs/core-utils)
  • 支持 CDN 引入 (unpkg, jsDelivr)
  • 提供 UMD、ES Module、CommonJS 等多种格式
  • 包含完整的类型定义文件 (index.d.ts)
  • 支持 tree-shaking 优化
  • 提供 package.json 元数据（keywords, homepage, repository, bugs等）
